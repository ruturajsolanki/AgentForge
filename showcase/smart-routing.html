The three-tier fallback chain

NIM's free tier rate-limits at 40 RPM. So we wrapped every primary call in a fallback chain that auto-translates model IDs across providers. When NIM hiccups, the next provider takes over — mid-stream — without the agent noticing.

The "holy moly" trick: speed shortcuts

Fallback is reactive — it kicks in after something fails. Smart routing is proactive. If a role only needs prose (docs, explanations, status updates), we skip NIM entirely and go straight to Groq. Same provider in your config — the router just knows better.

Live, token-by-token, on a WebSocket

Every agent's LLM call is now a stream. Each chunk is fanned out over a WebSocket as an agent.code event, then a per-agent live panel in the UI assembles it in real time. No more "did anything happen?" — you watch the code grow.

// backend/app/executor/base_agent.py
async def llm(self, prompt: str, system=None) -> str:
    routed = model_router.resolve(self.role)
    await self.emit("agent.code", {
        "phase": "start",
        "provider": routed.provider,
        "model": routed.model,
        "task": self.current_task,
    })
    async for chunk in provider.stream(model=routed.model, messages=msgs):
        await self.emit("agent.code", {
            "phase": "chunk",
            "delta": chunk,
            "seq": seq,
        })
    await self.emit("agent.code", {"phase": "end", ...})
